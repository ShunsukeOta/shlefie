# UI / MVP 定義（モバイルファースト・最小構成）

## 前提
- モバイル優先
- ミニマルかつシンプル
- 最短で最小限の機能を持たせて動くもの

## UI 方針
- モバイルファーストであること
- ベースカラーは `#222` / `#f9f9f9` の白黒
- 効果的な箇所のみ、コントラストの近い色を使用
- UIや文字サイズは全体的に小さめにし、画面内の情報量を増やす
- 角丸は `4px` までに抑える（効果的な箇所や主要メニューには使用可）
- iPhoneアプリのようなUI設計

## ナビゲーション
- 画面下に角丸50%のグローバルメニューを設置
- 左から `本棚` / `登録` / `マイページ`
- 設定画面への遷移は、マイページ右上の設定アイコン
  - もしくは全画面共通で右上に表示

## MVP 必要ページ
- 本棚
- 登録
- マイページ
- 設定

## 本棚ページ要件
- 登録している本棚を表示
- 表示形式は `グリッド`（2 / 3 / 4列の切替）または `リスト`
- 簡単なフィルタや並び順の調整が可能
- 各本のステータスを4段階程度で調整可能
  - 欲しい / 積読 / 読了 / 読書中
- タグチップのような表示で見やすくする

## 登録ページ要件
- APIを使用してリアルタイム検索
  - `本の名前` と `著者` の2パターンで検索
- 情報がある場合、書籍を選択すると自動入力
  - サムネイル
  - 発行年
  - 巻数などの基本情報

## マイページ要件
- 登録している冊数の表示
- 登録している本の進捗の総合
- 1週間 / 1か月 / 1年 のグラフで summary 表示
- 簡単なログの表示

## 設定ページ要件
- 最小機能としてエクスポート / インポート
  - JSON もしくは CSV
- 将来に向けた設定項目
  - プロフィール画像
  - プロフィール文
  - フォロー / フォロワー
  - 本棚の公開設定
  - shelf 共有設定
